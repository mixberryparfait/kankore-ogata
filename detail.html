大型建造も　通常建造　開発　と同じく　テーブル決定　⇒　テーブル内のエントリ決定　の２フェーズになってます

テーブル決定

  • 各資源 2400/1050/2800/2800 を超えると 空母テーブルに入り始めて 3600/1950/4200/5200 を超えると空母テーブル確定
  • 空母テーブルに入らなかった場合
    各資源 2240/2940/4400/1050/20 を超えると 武蔵テーブルに入り始めて 3360/5460/6600/1950/20 を超えると武蔵テーブル確定
  • 武蔵テーブルにも入らなかった場合
    各資源 1920/2240/3040/910 を超えると 大和テーブルに入り始めて 2880/4160/4560/1690 を超えると大和テーブル確定
  • 以上どこにも入らなければ最低値テーブル

基本的には乱数にしてもメリットがないので入りたいテーブル確定分以上資源を盛る
戦艦テーブルに入りたい場合はボーキを空母テーブル以下に
大和テーブルに入りたい場合は開発資材を 1 にするだけで調整できます

エントリ決定

空きドック依存の始点乱数 => 資源定数依存の乱数分巻戻った位置のエントリ一を選ぶ

始点乱数

空きドック 0 - 3 に応じて始点が以下の範囲から一様乱数で選ばれる

  • 空き 0: 3 ～ 99
  • 空き 1: 1 ～ 99
  • 空き 2: 1 ～ 95
  • 空き 3: 1 ～ 91

空きドックが多いほど前半から始まりやすい

巻き戻り乱数

始点位置から 0 ～ 資源定数(後述) の乱数分巻戻る (資源定数分確定でま戻るわけではない)

(例)始点 50 で資源定数 10 の場合 0～-9 されて 41～50 からランダムで選ばれる

資源定数が 0 なら空きドックの始点がそのまま建造結果になる

エントリが 0 以下になった場合 2 から再びおりかえす
なので 0 番エントリにあたることは絶対にない？？？

(例)始点 10 で資源定数 15 の場合
0～-14 巻き戻されて -4 ～ 10 が選ばれる
0 ～ -4 の5個にあったときは 2～6 に置き換わる

基本的には始点で後半を引いても資源定数が大きいほど前半のエントリに行きやすくなる

資源定数

  • 空母テーブル
    (燃料 - 3000) * 0.003 + (弾薬 - 2000) * 0.003 + (鋼材 - 4000) * 0.004 + (ボーキ - 5000) * 0.005 + (開発資材 - 50) * 0.1

    資源定数 +1 するのに 燃料 333, 弾薬 333, 鋼材 250, ボーキ 200, 開発資材 10

  • 武蔵テーブル
    (燃料 - 3500) * 0.003 + (弾薬 - 4500) * 0.005 + (鋼材 - 5500) * 0.004 + (ボーキ - 2200) * 0.002 + (開発資材 - 60) * 0.2

    資源定数 +1 するのに 燃料 333, 弾薬 200, 鋼材 250, ボーキ 500, 開発資材 5

  • 大和テーブル
    (燃料 - 2500) * 0.002 + (弾薬 - 3000) * 0.003 + (鋼材 - 4000) * 0.003 + (ボーキ - 1800) * 0.002 + (開発資材 - 40) * 0.2

    資源定数 +1 するのに 燃料 500, 弾薬 333, 鋼材 333, ボーキ 500, 開発資材 5

  • その他テーブル
    (燃料 - 2000) * 0.002 + (弾薬 - 2500) * 0.003 + (鋼材 - 3000) * 0.002 + (ボーキ - 1500) * 0.002 + (開発資材 - 40) * 0.2

    資源定数 +1 するのに 燃料 500, 弾薬 333, 鋼材 500, ボーキ 500, 開発資材 5

テーブル確定最低値より集めやすさと係数に応じてある程度資源定数（巻き戻り数）をふやして前半に偏らせた方が効率がいい
開発資材の価値は他の資源 数十程度
